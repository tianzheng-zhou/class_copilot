# class_copilot（听课助手）— 需求文档

> 版本：v0.4.0（ASR 方案切换至讯飞）  
> 日期：2026-03-14  
> 状态：需求已对齐，待进入开发

---

## 1. 项目概述

### 1.1 项目背景

大学思政课（思想政治理论课）中，教师经常会在课堂上提出问题，学生积极回答可以获得 **课堂表现分**。然而：

- 思政课内容涉及面广，学生不一定能快速组织出完整、得体的回答
- 课堂节奏快，思考时间有限
- 回答需要符合课程语境，措辞得当

本项目旨在构建一个 **实时听课辅助工具**，帮助学生在课堂上快速获取高质量的答题参考，提升课堂表现。

### 1.2 项目名称

| 中文名 | 英文名 |
|--------|--------|
| 听课助手 | **class_copilot** |

### 1.3 目标用户

- 大学生（主要场景：思政课，同时支持中文和英文授课课程）
- 后续可扩展至其他课程

### 1.4 运行环境与课堂场景

| 项目 | 说明 |
|------|------|
| 操作系统 | Windows |
| Python 版本 | 3.12.4 |
| 网络环境 | 校园网（稳定） |
| 硬件约束 | 笔记本电脑，可能无外接电源，**禁止使用本地模型**以节省电量 |
| 麦克风 | 笔记本内置麦克风阵列 |
| 课堂时长 | ~90 分钟 |
| 教室环境 | 安静，无明显噪音干扰 |
| 课堂人数 | 一般只有老师在说话；偶尔有同学做 Presentation |
| 提问窗口期 | 老师提问后约 10 秒内需要举手 |

## 2. 核心功能需求（MVP）

### 2.1 实时语音识别 + 说话人分离

| 项目 | 说明 |
|------|------|
| 功能 | 实时采集课堂音频，将语音转为文字，并区分不同说话人 |
| 输入 | 笔记本内置麦克风阵列音频流 |
| 输出 | 带说话人标签的实时文字转写结果 |
| 关键指标 | 延迟 ≤ 3秒，识别准确率 ≥ 90% |
| 语言支持 | 中英 + 202种方言免切识别（`lang=autodialect`） |
| 英文翻译 | 英文授课时，使用 LLM 将转写文本翻译为中文，支持英文原文 + 中文翻译双语展示 |
| 录音启动 | 手动点击开始 |
| 持续录音 | 支持 90 分钟连续录音转写（单次最大 8 小时） |
| 方案 | **科大讯飞 实时语音转写大模型版**，云端处理，原生支持说话人分离 |
| 协议 | WebSocket（`wss://office-api-ast-dx.iflyaisol.com/ast/communicate/v1`） |
| 音频参数 | 采样率 16kHz、16bit、单声道；建议每 40ms 发送 1280 字节 |
| 音频格式 | pcm / speex / opus |
| 教育优化 | `pd=edu` 教育领域专用参数 |
| 说话人分离 | `role_type=2` 开启实时角色分离，配合声纹注册使用 |

### 2.2 说话人识别（声纹管理）

| 项目 | 说明 |
|------|------|
| 功能 | 识别并标记教师的发言，过滤非教师发言 |
| 首次使用 | 开启盲分模式（`role_type=2`）自动分离说话人，用户手动点击标记"这个是老师" |
| 声纹注册 | 标记后录制老师 10s~1min 音频片段，调用讯飞声纹注册 API 获取 `feature_id`，下次上课自动匹配 |
| 声纹匹配 | 转写时携带 `feature_ids` 参数，云端自动匹配说话人身份；`eng_spk_match=1` 确保结果角色全部来自声纹库 |
| 声纹 API | 注册：`POST /res/feature/v1/register`；更新：`POST /res/feature/v1/update`；删除：`POST /res/feature/v1/delete` |
| 数据管理 | `feature_id` 按「课程名 + 老师名称」关联存储（本地 SQLite），一般一门课一个老师 |
| 存储位置 | 声纹特征存储在讯飞云端，`feature_id` 与课程映射关系存储在本地 |

### 2.3 LLM 转写过滤

| 项目 | 说明 |
|------|------|
| 功能 | 控制哪些说话人的转写文本发送给 LLM 进行问题检测和答案生成 |
| 默认模式 | 仅发送教师（已标记音色）的转写文本 |
| 可选模式 | 发送所有说话人的转写文本（适用于同学做 Presentation 等场景） |
| 切换方式 | 界面上提供切换按钮 / 快捷键 |

### 2.4 问题检测

| 项目 | 说明 |
|------|------|
| 功能 | 从转写文本中自动识别教师提出的问题 |
| 触发方式 | **自动检测 + 手动快捷键补充触发**（双模式） |
| 自动触发条件 | 检测到疑问句式、关键词（如"同学们想一想""谁来回答"等），由 LLM 辅助判断 |
| 手动触发 | 快捷键手动标记当前内容为提问，作为自动检测的补充 |
| 提醒方式 | **Windows 系统通知**（不使用声音提醒，避免课堂暴露） |
| LLM | 阿里云百炼 qwen3.5-flash |

### 2.5 智能答案生成

| 项目 | 说明 |
|------|------|
| 功能 | 基于检测到的问题，调用大语言模型生成参考答案 |
| 答案风格 | **口语化**，像学生口头回答的风格，自然流畅 |
| 立场要求 | prompt 中简要提及符合正确价值观即可，模型本身三观端正 |
| 上下文来源 | 完全依赖实时转写内容 + 课程名称，无需额外知识库 |
| 输出模式 | **简洁版 + 展开版** 两个参考答案（可在设置中开关） |
| 引用出处 | 不需要，保持自然口语风格 |
| 语言 | 跟随授课语言；英文授课时答案同时提供英文和中文翻译 |
| 上下文增强 | 可选导入课件/PPT 作为答案生成上下文（非必需） |
| 响应时间 | ≤ 3秒（qwen3.5-flash 速度快） |
| LLM | 阿里云百炼 qwen3.5-flash（MVP），后续可升级高精度模型 |

### 2.6 实时课堂提问（主动问答）

| 项目 | 说明 |
|------|------|
| 功能 | 用户可以主动输入问题，基于当前课堂上下文（转写内容）获取 LLM 回答 |
| 使用场景 | 课堂上遇到不理解的内容，实时向 AI 提问获取解释 |
| 上下文 | 自动携带当前课堂的转写记录作为上下文 |
| 输入方式 | 悬浮窗中的输入框，支持快捷键呼出 |
| LLM | 阿里云百炼 **qwen3.5-plus**（高级模型，确保回答深度和质量） |
| 历史记录 | 提问和回答记录随课堂记录一起保存 |

### 2.7 课堂记录与存储

| 项目 | 说明 |
|------|------|
| 功能 | 保存每节课的完整音频、转写记录和问答记录 |
| 音频存储 | 完整课堂录音，本地保存 |
| 转写存储 | 完整转写文本（含说话人标签和时间戳） |
| 问答存储 | 检测到的问题 + 生成的答案 + 用户主动提问记录 |
| 组织方式 | 按日期 / 课程名归档 |
| 软件内查看 | 支持在软件内浏览历史课堂记录 |
| 导出功能 | 支持导出为 Markdown / 文本文件 |
| 数据清理 | 不自动清理，永久保留 |

### 2.8 用户界面

| 项目 | 说明 |
|------|------|
| 框架 | **PyQt6** 桌面应用 |
| 形态 | 悬浮窗模式，支持置顶、最小化到系统托盘 |
| 默认位置 | 屏幕右下角 |
| 默认大小 | 紧凑窗口（约 400×500px），可拖拽调整 |
| 主题 | **深色模式**（课堂上更不显眼），后续可选浅色 |
| 核心展示区 | 实时转写文本（教师发言高亮）、检测到的问题、参考答案（简洁版/展开版）、主动提问区 |
| 隐蔽模式 | 快捷键快速隐藏/显示，最小化到托盘 |
| 交互 | 快捷键操作为主，鼠标操作为辅 |

## 3. 非功能需求

| 维度 | 要求 |
|------|------|
| **隐蔽性** | 深色界面，支持快捷键操作，快速隐藏/显示，系统通知提醒 |
| **低功耗** | 全部使用云端模型，不使用本地模型，降低电量消耗 |
| **响应速度** | 端到端（语音→问题→答案）总延迟目标 ≤ 6秒（ASR ≤ 3s + LLM ≤ 3s） |
| **持久运行** | 支持连续 90 分钟稳定运行 |
| **可扩展性** | 架构设计应支持后续功能扩展（见第 5 节）和模型升级 |
| **数据持久化** | 音频、转写、问答记录全量本地保存，永不自动删除 |
| **断网处理** | 网络中断时继续录音保存音频，暂停转写，界面提示离线状态，网络恢复后自动重连 |

## 4. 技术选型（确定方案）

| 模块 | 方案 |
|------|------|
| 语音识别（ASR） | **科大讯飞 实时语音转写大模型版**，WebSocket 协议，原生说话人分离 + 教育领域优化 |
| 说话人识别 | **科大讯飞云端声纹注册 API**（注册/更新/删除），`feature_id` 映射关系本地存储 |
| 问题检测 | 规则引擎 + 阿里云百炼 qwen3.5-flash 辅助判断 |
| 答案生成 | 阿里云百炼 qwen3.5-flash（自动答案） |
| 主动问答 | 阿里云百炼 **qwen3.5-plus**（用户主动提问，高质量回答） |
| 英文翻译 | 阿里云百炼 qwen3.5-flash（实时翻译转写文本） |
| 前端界面 | PyQt6 悬浮窗桌面应用 |
| 语言 | Python 3.12.4 |
| 数据存储 | SQLite（结构化数据）+ 本地文件系统（音频文件） |
| API Key 存储 | 加密存储（本地），分别管理讯飞 ASR 和阿里云 LLM 的密钥 |

| 方向 | 说明 |
|------|------|
| 📝 课堂笔记自动生成 | 基于转写文本，自动整理结构化课堂笔记 |

| 📱 移动端支持 | 手机端 App，更隐蔽更便捷 |
| 🧠 高精度模型升级 | 切换到更强大的 LLM 以提升答案质量 |
